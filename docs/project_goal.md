## Project Goal

AI coding agents are quickly gaining adoption, but the next unlock will come from running many agents in parallel—not just one at a time. The current user experience is not ready for this shift. Developers are increasingly acting as “AI babysitters,” micro-managing interactions instead of focusing on higher-level engineering goals.

Our project aims to flip that dynamic:
- Make the developer more like a manager than a babysitter.
- Provide a workflow for agent interaction that feels like managing email—organized, asynchronous, and scalable.
- Enable developers to effectively manage 10+ parallel agents without losing track of tasks, context, or quality.

The sweet spot for this system starts after todo list/backlog management and ends before becoming git management.  There are already better tools for those jobs. Tinstar starts at the prompt—where work is initiated—and carries all the way through QA, providing a structured yet lightweight scaffold that makes agentic coding workflows visible, coordinated, and manageable at scale.
